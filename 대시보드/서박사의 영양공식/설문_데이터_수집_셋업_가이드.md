# 설문 데이터 수집 — 사장님 셋업 가이드 (Apps Script 버전)

**소요 시간:** 약 7분
**목표:** 사용자가 설문을 끝낼 때마다 응답과 추천 결과가 사장님 구글 시트에 자동으로 한 줄씩 쌓이게 만든다.

---

## 한눈에 보기

```
사용자 설문 완료
  → nutriformula.co.kr 서버 (Vercel)
  → 사장님 계정의 Apps Script 웹훅
  → 사장님 구글 시트에 1줄 추가 ✅
```

만들 것:
1. 구글 시트 (데이터가 쌓이는 곳)
2. Apps Script 웹훅 (시트에 기록하는 작은 로봇)
3. Vercel 환경변수 3개

> 💡 왜 Google Cloud / 서비스 계정 방식에서 바뀌었나요?
> 구글이 2024년부터 새 프로젝트에 서비스 계정 키 생성을 자동 차단합니다(`iam.disableServiceAccountKeyCreation`).
> Apps Script 방식은 사장님 본인 계정 권한으로 돌기 때문에 키 발급 자체가 필요 없고, 관리할 비밀도 훨씬 적습니다.

---

## 1단계. 구글 시트 만들기

1. https://sheets.google.com → **"빈 스프레드시트"**
2. 제목: `서박사의 영양공식 - 설문 응답`
3. 하단 시트 탭 이름을 `Sheet1` → **`Responses`** 로 변경 (더블클릭)
4. A1셀을 선택한 뒤 아래 18개 헤더를 **그대로 복사해서 붙여넣기**:

```
timestamp	session_id	device	age_group	gender	height	weight	bmi	symptoms	goals	lifestyle	current_supplements	conditions	family_history	recommendations	persona_id	ip_hash	ua_hash
```

(탭으로 구분되어 있어 A1에 붙여넣으면 자동으로 A1~R1 18칸에 들어갑니다)

---

## 2단계. Apps Script 웹훅 배포

1. 같은 시트에서 상단 **확장 프로그램 → Apps Script** 클릭
2. 새 탭이 열리면 기본으로 보이는 `function myFunction() {}` 코드를 전부 지움
3. 제가 작성한 `tools/apps_script_webhook.gs` 파일을 열어 내용 전체를 복사 → Apps Script 편집창에 붙여넣기
4. **`SECRET` 값을 바꿉니다.** 상단 5번째 줄 근처:

   ```javascript
   const SECRET = 'CHANGE_ME_TO_A_LONG_RANDOM_STRING';
   ```

   이 문자열을 **아무도 모르는 긴 랜덤 문자열**로 교체. 예:

   ```javascript
   const SECRET = 'seoboksa-webhook-2026-7x9qK2mL8vN4pR6tY3uW';
   ```

   나중에 이 값을 Vercel에도 그대로 넣어야 하니, 메모장에 복사해두세요.

5. 상단 플로피 아이콘(💾) 또는 `Ctrl+S`로 저장. 프로젝트 이름을 묻는 창이 뜨면 `seoboksa-webhook` 같은 이름으로 저장.

6. 우측 상단 **배포 → 새 배포** 클릭
   - 좌측 상단 톱니 아이콘 → **"웹 앱"** 선택
   - 설명: `seoboksa survey ingest`
   - **다음 사용자 인증 정보로 실행:** `나 (saeronjhk@gmail.com)`
   - **액세스 권한이 있는 사용자:** `모든 사용자`
   - **"배포"** 클릭

7. 첫 배포 시 권한 승인 팝업이 뜹니다:
   - "액세스 승인" → 사장님 계정 선택
   - "이 앱은 Google에서 확인하지 않았습니다" 경고 → 좌측 하단 **"고급"** → **"seoboksa-webhook(안전하지 않음)으로 이동"**
   - "허용" 클릭 (본인이 만든 본인 계정 스크립트라 안전)

8. 배포 성공 화면에서 **웹 앱 URL**을 복사:
   ```
   https://script.google.com/macros/s/AKfycb.......................exec
   ```
   이 URL도 메모장에 복사.

## 3단계. Vercel에 환경변수 등록

1. https://vercel.com/dashboard → `nutriformula` 프로젝트 → **Settings → Environment Variables**
2. 아래 3개를 **Production / Preview / Development 모두 체크**하고 추가:

| Key | Value |
|-----|-------|
| `SHEETS_WEBHOOK_URL` | 2단계 8번에서 복사한 웹 앱 URL |
| `SHEETS_WEBHOOK_SECRET` | 2단계 4번에서 설정한 SECRET 값 (Apps Script와 **완전히 동일**해야 함) |
| `ANALYTICS_IP_SALT` | 아무 랜덤 긴 문자열. 예: `seoboksa-ipsalt-2026-a4b8c3d7e2f1` |

3. **Deployments** 탭 → 최신 배포 우측 `...` → **Redeploy** (환경변수는 재배포해야 적용됨)

## 4단계. 작동 확인

1. **헬스체크 (웹훅 살아있는지):** 브라우저 주소창에 Apps Script 웹 앱 URL을 그대로 붙여넣고 엔터. 아래처럼 뜨면 정상:
   ```json
   {"ok":true,"service":"seoboksa-survey-webhook"}
   ```

2. **실전 테스트:** https://nutriformula.co.kr 접속 → 하단에 **"🔒 서비스 품질 개선을 위해..."** 배너 확인 → 설문을 끝까지 진행해 추천 결과 화면까지 도달

3. 구글 시트 `Responses` 탭으로 이동 → 2행에 새 데이터가 들어왔는지 확인

1행이 안 들어오면:
- Vercel → 프로젝트 → **Logs** 탭에서 `[survey-submit] error:` 검색
- Apps Script 편집기 → **실행** 메뉴 → **실행** 로그 확인
- 에러 메시지를 그대로 저한테 보내주세요. 바로 진단합니다.

---

## SECRET이 뭐 하는 거예요?

웹훅 URL은 '공개' 배포라 URL을 아는 사람은 누구나 POST를 쏠 수 있습니다. SECRET이 없으면 장난 데이터가 섞일 수 있어요. 그래서 Vercel 서버에서 요청을 보낼 때마다 SECRET을 같이 보내고, Apps Script가 일치하지 않으면 거부하도록 했습니다. **Vercel 값과 Apps Script 값이 반드시 똑같아야 합니다.**

## 자주 묻는 질문

**Q. 완전 무료인가요?**
A. 네. Apps Script 웹앱은 하루 20,000 요청까지 무료, 설문 수천 건도 거뜬합니다.

**Q. SECRET을 바꾸고 싶으면?**
A. Apps Script 편집기에서 SECRET 값만 바꾸고 **"배포 → 배포 관리 → 편집(연필 아이콘) → 새 버전 → 배포"** 하면 됩니다. URL은 그대로 유지됩니다. 그 다음 Vercel 환경변수 `SHEETS_WEBHOOK_SECRET`도 똑같이 수정 + 재배포.

**Q. 개인정보가 사장님한테 흘러 들어오진 않나요?**
A. 이름·이메일·전화번호는 애초에 수집하지 않습니다. IP/UA는 해시된 16자만 기록돼 복원 불가능합니다. 건강 정보는 익명 세션 ID로만 묶여 있어 특정 개인을 역추적할 수 없습니다.

**Q. 데이터를 나중에 어떻게 활용하나요?**
A. 구글 시트에서 피벗 테이블/차트로 바로 보거나, CSV로 내보내서 저한테 주시면 분석 리포트를 만들어 드립니다.
